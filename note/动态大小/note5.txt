一、先定结论：Rust 里只有两大类 DST（动态大小类型）
所有 DST 共同特点：
编译期无法确定固定大小
不能单独在栈上直接声明变量
必须借指针 / 智能指针包裹使用：& / Box / Rc / Arc
两类 DST：
原生容器 DST：str、[T]
Trait 对象 DST：dyn Trait
二、三类关键类型 完整对比 + 内存布局
1. 普通定长类型（非 DST）
例：i32、bool、String、固定大小结构体
编译期尺寸固定
普通单指针即可管理
内存：只有「数据指针」
2. 原生 DST：str / [T]（字符串、切片）
✅ 唯一、确定的具体类型，不是 trait✅ 无多态、无虚表、无动态分发✅ 胖指针组成：数据指针 + 长度
内存布局（&str / &[T]）
plaintext
+-------------+
| 数据指针    |  指向堆/内存里的真实内容
+-------------+
| 长度(usize) |  记录元素个数/字节数
+-------------+
为什么不需要 dyn？
本身就是唯一具体类型，不存在「多种实现」，不需要运行时动态匹配。
合法：&str、Box<str>、Rc<[u8]>
非法：&dyn str、dyn [i32]
3. Trait DST：dyn Trait
❌ Trait 本身不是具体类型，是行为约束✅ 代表任意实现该 Trait 的不同类型（多态）✅ 需要 vtable 做运行时动态方法分发✅ 胖指针组成：数据指针 + 虚表指针 (vtable)
内存布局（&dyn Trait / Box<dyn Trait>）
plaintext
+-------------+
| 数据指针    |  指向具体实例：Cat / Dog / ...
+-------------+
| vtable指针  |  指向方法虚表，运行时查方法
+-------------+
为什么必须加 dyn？
用来显式告诉编译器：
这不是普通 trait 约束，而是需要 vtable 的 trait 对象、运行时多态。
合法：&dyn Animal、Box<dyn Animal>
非法：&Animal、Box<Animal>
三、一张表彻底分清
表格
类型	类别	是否具体类型	胖指针额外信息	是否需要 dyn	核心用途
i32 / 结构体	定长类型	是	无（单指针）	否	常规数据存储
str	原生 DST	是（唯一）	长度	❌ 不需要	字符串借用
[T]	原生 DST	是（唯一）	长度	❌ 不需要	数组切片
dyn Trait	Trait DST	否（抽象）	vtable 指针	✅ 必须加	多态、异构集合
四、回答你最初的灵魂问题
str 是具体原生类型，只是尺寸不固定 → 用「数据指针 + 长度」管理，不需要 dyn
Trait 是抽象行为约束，要容纳多种不同结构体 → 必须用 dyn 标记 + vtable 实现多态
关键一句话区分
&str ：我就是字符串本身，只是长度不定
&dyn Trait ：我是一层抽象壳，里面可以装任何实现这个特征的类型
五、延伸：语法规则极简记忆
只要是具体类型：str / [T] → 直接 &类型
只要是trait 多态对象 → 强制 &dyn 特征
dyn 全程只服务于 trait，和普通 DST 毫无关系。
