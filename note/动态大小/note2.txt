一、dyn Trait 完整核心梳理 + 写法规范 + 静态 / 动态分发对比 + 使用场景 + 对象安全
1. 核心定义
Trait：只是trait 约束 / 特征，本身是DST 动态大小类型，不能单独当类型用；
dyn Trait：trait 对象类型，代表「任意实现了该 trait 的未知具体类型」；
必须搭配指针 / 智能指针使用：
rust
运行
// 合法（trait 对象，动态多态）
&dyn Trait
Box<dyn Trait>
Rc<dyn Trait>
Arc<dyn Trait>

// 非法！新版 Rust 直接报错
&Trait
Box<Trait>
2. 两个核心模式：静态分发 vs 动态分发
① 静态分发：泛型 + Trait 约束
写法：
rust
运行
// T 是编译期确定的具体类型
fn foo<T: Trait>(item: T) {
    item.method();
}
特点：
编译期单态化：每种不同 T 生成一份专属代码；
方法调用直接跳转，无运行时开销；
类型编译期确定，不能存不同类型到同一个集合；
无 vtable、无胖指针。
② 动态分发：dyn Trait + 指针
写法：
rust
运行
// 运行期才知道具体是什么类型
fn foo(item: &dyn Trait) {
    item.method();
}
特点：
运行时通过 vtable 虚表 查找方法；
有微量运行时开销；
统一 dyn Trait 类型，可以存放不同实现类；
内部是胖指针：数据指针 + vtable 指针。
3. 为什么必须写 dyn？（关键区分歧义）
假设允许写 Box<Trait>：
编译器分不清：
你要「泛型约束的简写」？
还是「trait 对象动态多态」？
Rust 设计哲学：关键行为必须显式
写 <T: Trait> → 静态、编译期、单态化
写 dyn Trait → 动态、运行时、trait 对象
4. 常用标准写法（直接抄就能用）
引用借用（最省开销）
rust
运行
trait Animal {
    fn speak(&self);
}

fn talk(animal: &dyn Animal) {
    animal.speak();
}
堆上 trait 对象（Box，最常用）
rust
运行
let cat: Box<dyn Animal> = Box::new(Cat);
let dog: Box<dyn Animal> = Box::new(Dog);

// 异构集合：存不同类型
let animals: Vec<Box<dyn Animal>> = vec![cat, dog];
Rc / Arc 共享所有权
rust
运行
use std::rc::Rc;
let a: Rc<dyn Animal> = Rc::new(Cat);
5. 关键限制：对象安全（Object Safety）
不是所有 Trait 都能变成 dyn Trait一个 trait 要能作为 trait 对象，必须满足：
所有方法必须有 &self / &mut self /self 接收者；
不能有泛型方法；
不能有 async fn（async 底层含泛型）；
❌ 错误示例（非对象安全）：
rust
运行
trait BadTrait {
    // 无 self：无法绑定实例，不能动态分发
    fn func();
    // 泛型方法：无法统一 vtable
    fn gen<T>(&self);
}
6. 如何选择：用 泛型 (T: Trait) 还是 dyn Trait？
选 泛型静态分发
追求极致性能，零运行时开销；
调用处类型固定、单一；
不需要异构集合。
选 dyn Trait 动态分发
需要统一类型容器：Vec<Box<dyn T>>；
类型在运行时才确定；
插件化、回调、异构组件场景；
不想代码单态化膨胀，减小二进制体积。
7. 补充：和你原文 DST 的关联
你书本原文关键点：
str、[T]、dyn Trait 都是 DST 动态大小类型；
DST 本身无法单独栈分配、无法确定大小；
所以必须包裹指针：
&str / Box<str>
&[T] / Box<[T]>
&dyn Trait / Box<dyn Trait>
逻辑完全统一：
所有 DST 类型，都必须通过指针 / 智能指针间接使用。
8. 极简记忆口诀
静态多态写 T: Trait，编译确定快；
动态多态写 dyn Trait，运行查表灵；
无 dyn 不能写 trait 对象，语法强制；
要存多种实现类，必用 Box<dyn 特征>；
无 self、泛型方法 → 不能做 dyn 特征对象。
