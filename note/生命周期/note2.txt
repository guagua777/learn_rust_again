下面用极简、好懂、贴合你 next 方法的例子，一步步吃透 '_ 匿名生命周期。
一、先回顾你的源码
rust
运行
fn next(&mut self) -> Next<'_, Self> where Self: Unpin;
核心：
&mut self 有一个隐式生命周期
Next 是带生命周期的泛型类型：struct Next<'a, S>
返回的 Next 必须和 self 的生命周期绑定
二、版本 1：完整手写生命周期（最原始、最好理解）
rust
运行
// 显式声明生命周期 'a
fn next<'a>(&'a mut self) -> Next<'a, Self> {
    // ...
}
语义：
借用 self 的生命周期是 'a，返回的 Next 结构体不能活得比 'a 更久，保证引用安全，不会悬垂。
三、版本 2：换成 '_ 匿名生命周期（标准库常用写法）
rust
运行
fn next(&mut self) -> Next<'_, Self> {
    // ...
}
完全等价于上面的 'a 版本。
'_ 的含义：
这里需要一个生命周期参数，不能省
让编译器自动推断，绑定到最近的引用（这里就是 &mut self）
不用你手动起名 'a / 'b / 'c，简化代码
四、关键区分：两种「省略」不要搞混
1. 普通引用的生命周期省略（日常最常见）
rust
运行
// 本来完整写法：fn demo<'a>(s: &'a str)
fn demo(s: &str) { }
函数参数里的引用，Rust 有生命周期省略规则，可以直接不写。
2. 结构体 / 泛型里不能直接省略，必须用 '_
rust
运行
// 报错！缺少生命周期参数
fn bad(&mut self) -> Next<Self> 

// 正确：用匿名占位符
fn good(&mut self) -> Next<'_, Self>
👉 区别：
函数参数引用：可以直接不写生命周期
泛型类型、结构体、关联类型：必须填参数，偷懒就用 '_
五、超简短小示例，直观感受约束
rust
运行
struct Wrap<'a>(&'a i32);

// 使用 '_
fn get_wrap(val: &i32) -> Wrap<'_> {
    Wrap(val)
}

fn main() {
    let mut x = 10;
    let w = get_wrap(&x);
    // 下面这行编译失败
    // x = 20; 
    // 原因：w 借用了 x，生命周期绑定，不能同时可变借用
}
这里 Wrap<'_> 自动绑定 val 的生命周期，和你 Next<'_, Self> 逻辑一模一样。
六、一句话总结
'_ 就是合法生命周期，叫「匿名占位生命周期」。
作用：需要传生命周期参数，但不想手动命名，交给编译器自动绑定。
你写的 Next<'_, Self> = 返回值生命周期和 &mut self 严格绑定。
标准库迭代器、异步 trait 里全是这种写法，属于 Rust 规范简洁写法。
