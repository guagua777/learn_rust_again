闭包为什么不能直接返回？核心就是：大小不确定 + trait 是动态类型
你猜的大小无法确定完全正确，这是最核心的原因，我用最通俗的方式给你讲透。
1. 先记住 Rust 的铁律
Rust 是静态类型语言，编译时必须知道所有变量 / 返回值的固定大小。函数的返回值，必须是编译期就能确定大小的类型。
2. 闭包的本质：每个闭包都是独一无二的类型
rust
运行
// 这两个闭包，类型完全不同！
let f1 = || println!("1");
let f2 = || println!("2");
每个闭包，编译器都会生成一个匿名、独一份的结构体
不同闭包，大小不一样
编译器根本无法提前知道：你要返回的闭包到底占多少内存
3. 闭包表现为 trait，加剧了这个问题
Rust 里闭包实现了三个 trait：
Fn
FnMut
FnOnce
trait 本身不是具体类型，没有固定大小。你可以把 trait 理解成一个 **“接口 / 约定”**，不是真实的内存布局。
所以你不能写：
rust
运行
// ❌ 错误：trait 对象没有固定大小，不能直接返回
fn make_closure() -> dyn Fn() {
    || println!("hello")
}
4. 那怎么返回闭包？（两种正确方案）
方案 1：使用 impl Trait（推荐，静态分发）
告诉编译器：我返回一个实现了该 trait 的具体类型，由编译器自动推导大小。
rust
运行
// ✅ 正确
fn make_closure() -> impl Fn() {
    || println!("hello")
}
这不是返回 trait，而是返回一个匿名具体类型
编译期确定大小，完美符合 Rust 规则
方案 2：使用 Box<dyn Trait>（动态分发）
把闭包放到堆上，栈上只存固定大小的指针。
rust
运行
// ✅ 正确
fn make_closure() -> Box<dyn Fn()> {
    Box::new(|| println!("hello"))
}
Box 是指针，大小固定（8 字节，64 位系统）
真正的闭包存在堆上，大小无所谓
总结
不能直接返回闭包 trait：因为 trait 没有固定大小，Rust 编译期无法确定内存布局。
每个闭包都是独一无一的类型，编译器无法统一表示。
解决方案：
优先用 impl Fn/FnMut/FnOnce（静态分发，性能高）
需要多态时用 Box<dyn Fn(...)>（堆分配，动态大小）
