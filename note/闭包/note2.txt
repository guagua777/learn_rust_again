Rust 返回闭包：3 种方式彻底讲清（无矛盾版）
我直接给你最清晰、最准确、最不绕弯的总结，你一看就能彻底分清什么时候行、什么时候不行。
先记住一句话
Rust 函数返回值必须有固定大小。闭包本身没有固定大小，所以不能直接裸返回，必须包装。
三种返回闭包的方式
1. impl Fn (...) → 正确 ✅（静态分发，首选）
rust
运行
fn make_closure() -> impl Fn() {
    || println!("hello")
}
这是什么？
不是返回 trait
是告诉编译器：返回一个 “实现了 Fn 的具体匿名类型”
编译器在编译时能确定大小
优点
最快，无开销
最简洁
最推荐
限制
函数内只能返回一种闭包，不能返回多个不同闭包
2. Box<dyn Fn (...)> → 正确 ✅（动态分发）
rust
运行
fn make_closure() -> Box<dyn Fn()> {
    Box::new(|| println!("hello"))
}
这是什么？
把闭包放到堆上
Box 是指针，大小固定（8 字节）
dyn Fn() 表示 “动态类型的闭包”
优点
可以返回多种不同闭包
适合需要多态、条件返回不同闭包的场景
缺点
轻微堆分配开销
动态 dispatch 略微慢一点
3. 直接返回 dyn Fn (...) → 错误 ❌（绝对不行）
rust
运行
// 错误！
fn make_closure() -> dyn Fn() {
    || println!("hello")
}
为什么不行？
dyn Trait 没有固定大小
Rust 无法在栈上安排内存
编译直接报错：the size for values of type ... cannot be known
最关键的一张对比表
表格
写法	是否合法	类型	大小固定？	性能	用途
impl Fn()	✅ 合法	静态具体类型	固定	最快	只返回一种闭包
Box<dyn Fn()>	✅ 合法	堆上动态 trait 对象	固定（指针）	快	需要返回多种闭包
dyn Fn()	❌ 非法	动态裸 trait	不固定	-	永远不要用
最容易混淆的两个点（99% 的人都卡在这里）
1. impl Trait ≠ dyn Trait
impl Fn()：具体类型，大小确定 ✅
dyn Fn()：trait 本身，大小不确定 ❌
2. 不是 “闭包不能返回”
而是：
不能直接返回裸闭包（无包装）
不能直接返回裸 trait（dyn Fn）
但：
可以返回 impl Fn
可以返回 Box<dyn Fn>
最终结论（背会这两句）
想直接简单返回闭包 → 用 impl Fn()
想返回多种不同闭包 → 用 Box<dyn Fn()>
永远不要直接写 -> dyn Fn()
